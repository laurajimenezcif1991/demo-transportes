Guía de Demo — Unio B2B

Para uso interno del equipo · Versión local en Windows

Esta guía explica cómo instalar y correr la app de Unio en modo demo en un computador Windows desde cero, y cómo realizar una presentación completa del pipeline a empresas clientes.

1
Preparación inicial (solo la primera vez)
1
Instalar Node.js
Ve a nodejs.org en el navegador. Descarga el instalador con la etiqueta LTS. Abre el archivo .msi y sigue el asistente (siguiente → siguiente → instalar). Cuando termine, reinicia el computador.

Para verificar, abre la terminal y escribe: 
node --version   # debe mostrar algo como v20.x.x
2
Abrir la terminal de Windows
Busca PowerShell en el menú Inicio → clic derecho → Ejecutar como administrador.
Alternativa: presiona Windows + R, escribe cmd y Enter.
3
Descomprimir el .zip del proyecto
Haz clic derecho sobre el archivo unio-demo.zip → Extraer todo... → elige una carpeta fácil de encontrar, por ejemplo:
C:\Users\TuNombre\Desktop\unio-demo

Dentro de esa carpeta debe aparecer una subcarpeta llamada unio-app.
4
Navegar a la carpeta en la terminal
En la terminal, escribe el siguiente comando con la ruta donde descomprimiste el zip: 
cd C:\Users\TuNombre\Desktop\unio-demo\unio-app
Tip: puedes arrastrar la carpeta unio-app desde el Explorador de archivos directamente a la ventana de la terminal — escribirá la ruta sola. Luego presiona Enter.

Para confirmar, escribe dir y Enter. Debes ver archivos como package.json.
5
Instalar las dependencias
npm install   # puede tardar 1–3 minutos, espera hasta que vuelva el cursor
Este paso solo es necesario la primera vez.
2
Levantar la app (cada vez que hagas demo)

Con la terminal abierta y ubicada en la carpeta unio-app, corre:

npm run dev

La terminal mostrará:

  VITE v5.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/

Abre Chrome o Edge, escribe http://localhost:5173 en la barra de direcciones y presiona Enter.

Importante: mantén la terminal abierta durante toda la demo. Si la cierras, la app deja de funcionar.
3
Las 5 vacantes demo

Todas aparecen en la pantalla de inicio "Tus vacantes". Cada una cubre una etapa distinta del pipeline.

Vacante Etapa activa Candidatos Qué mostrar
Recepcionista Scoring IA 15 Semáforo de resultados, no negociables, hoja de vida
Auxiliar de Bodega Pre-screening IA 15 Entrevista IA, resumen generado, análisis de perfil
Analista de TH Entrevistas 10 Formularios de entrevista, notas del evaluador
Jefe de Finanzas Evaluaciones 6 Prueba psicológica, radar de competencias, prueba técnica
Gerente de Ventas Finalistas 3 Grid comparativo de finalistas, scorecard completo
4
Guión de demo recomendado (15–20 min)
Dashboard: señala las 5 vacantes, el progreso por etapa y los indicadores generales.
Recepcionista → Scoring IA: muestra el semáforo de candidatos. Abre un perfil y señala no negociables, logros y señales por validar. Haz clic en Hoja de vida — se abre en pestaña nueva como un documento real.
Auxiliar de Bodega → Pre-screening IA: muestra cómo la IA entrevistó a los candidatos. Abre un perfil y despliega el acordeón de Pre-entrevista IA.
Analista de TH → Entrevistas: muestra formularios con respuestas del candidato. Señala la progresión del embudo (menos candidatos que en etapas anteriores).
Jefe de Finanzas → Evaluaciones: despliega la prueba psicológica (radar, veredicto, gaps). Señala quiénes contestaron la prueba técnica y quiénes no. Demo interactiva: selecciona 3 candidatos → "Pasar etapa" → Finalistas se desbloquea en tiempo real.
Gerente de Ventas → Finalistas: muestra el grid comparativo de los 3 finalistas. Abre el perfil de uno y recorre todos los acordeones (scoring → prescreening → entrevistas → psico → técnica).
5
Función "Pasar etapa" — demo interactiva

La función más impactante para mostrar en vivo. Úsala en Jefe de Finanzas → Evaluaciones:

1
Selecciona exactamente 3 candidatos usando los checkbox de la izquierda.
2
Haz clic en el botón "Pasar etapa".
3
Los candidatos aparecen en Finalistas con el chip "Pendiente" en gris.
4
El sidebar izquierdo desbloquea Finalistas automáticamente. Navega ahí para mostrar la vista comparativa.
Tip: si seleccionas menos de 3 o más de 3, aparece un modal explicativo. Es parte del flujo — muéstralo para reforzar la lógica de negocio.
6
Resetear la demo

Para volver al estado inicial antes de una nueva demo:

1
En Chrome o Edge, presiona F12 para abrir las herramientas de desarrollo.
2
Ve a la pestaña Application (si no la ves, haz clic en »). Luego en el panel izquierdo: Local Storage → http://localhost:5173.
3
Elimina todas las entradas que comiencen con mock_stage_ (clic derecho → Delete).
4
Recarga la página con F5. Todo vuelve al estado original.
7
Preguntas frecuentes
¿Necesito internet para la demo?
No. Una vez que corriste npm install por primera vez, todo funciona sin conexión. Solo necesitas la terminal abierta.
¿Los datos son reales?
No. Todos los candidatos, vacantes y empresas son completamente ficticios e inventados para propósitos de presentación.
¿Puedo hacer la demo desde proyector o pantalla externa?
Sí. Conecta el proyector antes de abrir el navegador y usa localhost:5173 en pantalla completa (F11).
¿Qué hago si npm install da error de permisos?
Cierra la terminal, ábrela de nuevo con clic derecho → Ejecutar como administrador y repite el comando.
La app no carga o da error, ¿qué hago?
Verifica que la terminal esté abierta y muestre el mensaje de Vite. Si no, navega a la carpeta unio-app con cd y corre npm run dev de nuevo. Para recargar forzado usa Ctrl + Shift + R.
El cliente pregunta por una funcionalidad que no está en demo, ¿qué hago?
Toma nota y avísale al equipo. No improvises mostrando rutas fuera del flujo de las 5 vacantes demo.
